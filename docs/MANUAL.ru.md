# О программе

[![Vinyller](assets/vinyller_main_themes.png)](assets/vinyller_main_themes.png)

Виниллер — это современный кроссплатформенный проигрыватель для локальной коллекции музыки с поддержкой всех 
основных аудиоформатов и возможностью чтения по трекам монолитных FLAC+CUE альбомов. 
Проигрыватель позволяет гибко настроить интерфейс и легко организовать фонотеку с помощью встроенного редактора тегов 
и создания собственной музыкальной энциклопедии. 

Проигрыватель поддерживает работу на современных операционных системах Windows, macOS, Linux. 
Минимальные системные требования — двухъядерный процессор и 4 Гб ОЗУ.

Посетить официальную страницу [Виниллер на GitHub](https://www.github.com/maxcreations/vinyller).

# Основные возможности проигрывателя

Этот раздел содержит описание основных функций проигрывателя Виниллер.

## Хранение пользовательских данных
Все существующие пользовательские файлы хранятся и обрабатываются **локально**. 
Файлы располагаются в папке `.vinyller` (`VINYLLER_FOLDER`) в домашнем каталоге пользователя, 
например `C:\Users\user_name\.vinyller`. 

Каталог может содержать:
- **Файлы проигрывателя** — настройки;
- **Файлы фонотеки** — кэш и обложки альбомов; файлы списка избранного, черного списка и «недоступного избранного»; данные об отложенных обновлениях фонотеки;
- **Файлы энциклопедии** — энциклопедия, изображения энциклопедии, резервные копии энциклопедии;
- **Файлы статистики** — списки с рейтингом композиций;
- **Файлы истории** — список истории воспроизведения; данные о последней открытой странице и очереди воспроизведения;
- **Файлы поисковых служб** — список поисковых служб, добавленных пользователем и список отключенных стандартных служб быстрого поиска.
- **Временные файлы** — файлы, создающиеся динамически перед окончательным сохранением в основные данные проигрывателя.

Список файлов может меняться в зависимости от настроек проигрывателя (отключение формирования статистики, истории воспроизведения и т.д.) и может расширяться в дальнейшем вместе с развитием проигрывателя.

## Приватность и сетевая активность

Виниллер — это приложение, ориентированное на локальную работу и уважающее приватность. Проигрыватель **не собирает телеметрию, не требует 
создания учетных записей и не отправляет данные о вашей фонотеке или истории прослушиваний на сторонние серверы**. 

Сетевая активность приложения строго ограничена функциями загрузки метаданных и инициируется только при использовании 
соответствующих инструментов. Ниже приведено подробное описание того, какие данные и куда отправляются:

* **Apple Music (iTunes Search API)**
  * **Где используется:** Во встроенном [редакторе метаданных](#редактор-метаданных) для поиска обложек и списков треков.
  * **Что отправляется:** При нажатии кнопки поиска проигрыватель отправляет только текст из строки запроса (например, *«Имя исполнителя Название альбома»*) на публичный сервер `itunes.apple.com`. При загрузке списка треков отправляется только цифровой идентификатор альбома (`collectionId`). Никакие данные о ваших локальных файлах не передаются.
* **LRCLIB (Тексты песен)**
  * **Где используется:** В [редакторе метаданных](#редактор-метаданных) для скачивания текстов песен.
  * **Что отправляется:** Проигрыватель обращается к открытому API `lrclib.net` и передает следующие теги запрашиваемой композиции: **Исполнитель**, **Название**, **Альбом** и **Длительность трека** в секундах. Эти данные необходимы для точного поиска текста.
* **Wikipedia**
  * **Где используется:** В [музыкальной энциклопедии](#энциклопедия) для автоматического заполнения описания статей и загрузки изображения статьи.
  * **Что отправляется:** При запуске поиска отправляется только текстовый поисковый запрос (название статьи) и выбранный язык для получения вводного текста статьи и URL-адреса заглавного изображения. Изображение скачивается напрямую по публичной ссылке.
* **Поисковые службы**
  * **Где используется:** При выборе пункта [контекстного меню](#контекстное-меню-и-действия) «Найти в интернете...» или при нажатии кнопки «Найти...» в [редакторе метаданных](#редактор-метаданных) и при редактировании статьи [энциклопедии](#энциклопедия).
  * **Как это работает:** Виниллер **не делает** сетевых запросов самостоятельно в этом режиме. Программа лишь формирует текстовую ссылку (например, подставляя имя артиста в URL Spotify или Google) и передает команду вашей операционной системе открыть эту ссылку в браузере по умолчанию.

**Загрузка обложек:** Все изображения (как для энциклопедии, так и для фонотеки) загружаются по прямым URL-адресам с использованием стандартных HTTP-запросов и сохраняются в ваш локальный каталог.

## Поддерживаемые форматы аудиофайлов
.mp3, .flac, .ogg, .wav, .m4a, .mp4, .wma, .aac, а также [виртуальное разделение FLAC+CUE](#парсинг-монолитных-flaccue-альбомов) альбомов на треки.

## Музыкальный центр

[![Vinyller](assets/favorites/favorites_music_center.png)](assets/favorites/favorites_music_center.png)

Музыкальный центр служит основной вкладкой, куда будут попадать добавленные в избранное композиции, альбомы, исполнители, композиторы, списки воспроизведения и папки. 

### Блок «Моя волна»
В основном блоке музыкального центра находятся кнопки «Моя волна» и «Случайный альбом».

- Кнопка «Моя волна» формирует очередь воспроизведения из случайных композиций на основе музыки, добавленной в избранное, и рейтинга прослушанной музыки (если включена опция формирования статистики).
- Кнопка «Случайный альбом» запускает воспроизведение случайного альбома из всей фонотеки.

### Как формируется очередь воспроизведения «Моя Волна»

Алгоритм формирования умной очереди **«Моя волна»** базируется на системе взвешенного случайного выбора, которая отдает приоритет вашим музыкальным предпочтениям. 

1. **Сбор предпочтений:** Приложение загружает ваши списки избранного (любимые треки, артисты, альбомы и жанры).
2. **Создание пула кандидатов:** Чтобы избежать подвисаний на огромных медиатеках, из всей базы случайным образом отбирается ограниченный пул треков для анализа. Размер пула регулируется константой `MY_WAVE_SAMPLE_POOL_SIZE` (по умолчанию 2000).
3. **Назначение весов (оценка):** Каждому треку из пула изначально дается базовый шанс попадания в очередь — `MY_WAVE_BASE_SCORE` (по умолчанию 1). Затем вес увеличивается при совпадениях с вашим избранным:
   - Сам трек находится в избранном: прибавляется `MY_WAVE_BONUS_FAV_TRACK` (+50)
   - Исполнитель трека в избранном: прибавляется `MY_WAVE_BONUS_FAV_ARTIST` (+30)
   - Альбом трека в избранном: прибавляется `MY_WAVE_BONUS_FAV_ALBUM` (+20)
   - Один из жанров трека находится в избранном: прибавляется `MY_WAVE_BONUS_FAV_GENRE` (+15, начисляется единожды)
4. **Взвешенная выборка:** Из оцененного пула отбираются треки. Вероятность попадания конкретного трека напрямую зависит от его итогового веса. Для гарантии того, что после фильтрации останется нужное количество треков, алгоритм выбирает элементы с запасом, умножая искомый лимит на `MY_WAVE_OVERSAMPLE_MULTIPLIER`.
5. **Фильтрация и лимиты:** Алгоритм очищает выборку от дубликатов, оставляя только уникальные записи. Процесс останавливается, когда очередь заполняется до лимита, заданного в `MY_WAVE_DEFAULT_LIMIT` (по умолчанию 50 треков).

### Добавление музыки в избранное
Для того чтобы добавить в избранное композицию, альбом, исполнителя, композитора, список воспроизведения или папку можно использовать несколько методов:
- **[Контекстное меню](#контекстное-меню-и-действия)** — вызвать контекстное меню на интересующей сущности с помощью нажатия правой кнопки мыши и выбрать соответствующий пункт меню;
- **Панель управления воспроизведением** — нажать кнопку избранного (по умолчанию — <kbd>♥</kbd>), для добавления текущей проигрываемой композиции в список избранных;
- **Панель инструментов текущей страницы** — нажать кнопку избранного (по умолчанию — <kbd>♥</kbd>) в верхней панели открытой страницы, для добавления текущей просматриваемой сущности (альбом, исполнитель, композитор, жанр, список воспроизведения или папка) в список избранных;

Удаление из списка избранного осуществляется обратным действием, т.е. повторным нажатием соответствующей кнопки или пункта меню.

### Недоступное избранное

[![Vinyller](assets/favorites/favorites_settings_unavailable_wnd.png)](assets/favorites/favorites_settings_unavailable_wnd.png)

Если по какой-либо причине элементы, добавленные в список избранного, стали недоступны (изменение расположения или удаление 
файлов, перенос пользовательских файлов настроек между устройствами или отключение накопителя и т.д.), то:
- Элементы скрываются из интерфейса вкладки «Музыкальный центр» и попадают в список «Недоступного избранного», которым можно управлять в **[окне настроек](#настройки-программы)** проигрывателя на вкладке **«Фонотека»**.
- При восстановлении доступа к файлам (с условием совпадения расположения) и перезапуске проигрывателя список недоступного будет автоматически очищен, а элементы будут восстановлены на вкладку «Музыкальный центр».


### Список популярных композиций
Если включена опция формирования [статистики прослушиваний](#рейтинг-и-чарты), то в данной вкладке также будет отображаться блок «Популярное в этом месяце», содержащий короткий список часто прослушиваемых композиций.

### Навигация и управление
- **Группировка и быстрая навигация:** Если в настройках включено отображение **интерактивных разделителей**, карточки на корневых страницах вкладок будут визуально разделены на смысловые блоки в зависимости от выбранного режима сортировки (по заглавной букве при алфавитной сортировке, по датам при сортировке по году выпуска, или по месяцам при сортировке по дате добавления). Клик по такому разделителю откроет **меню быстрых переходов** к другим разделителям страницы.
- **Случайные предложения:** При наличии 20 и более элементов на вкладке и включенной соответствующей опции в настройках, в верхней части вкладок может появляться рекомендательный блок со случайной подборкой элементов для прослушивания.
- **Быстрое воспроизведение:** Клик по интерактивной кнопке обложки на карточке моментально запускает воспроизведение выбранной сущности.
- **Переход к внутренним страницам:** Клик по карточке исполнителя, композитора, жанра или папки открывает список альбомов выбранной сущности. Клик по карточке альбома открывает детальную страницу с трек-листом, откуда можно запустить воспроизведение как всего релиза, так и отдельных композиций.
- **Кнопка <kbd>←</kbd> (Назад):** позволяет возвращаться на шаг назад в рамках каждой вкладки по отдельности.

В зависимости от контекста, панели управления страницами могут содержать:
- **Кнопка сортировки:** позволяет упорядочить элементы по различным критериям (по алфавиту, году выпуска, дате добавления и другие варианты, в зависимости от контекста).
- **Кнопка вида:** позволяет переключать стиль отображения элементов на странице (сетка, крупная плитка, мелкая плитка, а в случаях просмотра исполнителей, композиторов и папок — карточки альбомов со списком композиций).
- **Кнопки действий:** позволяет добавлять в избранное, воспроизводить и т.д. просматриваемую сущность.
- **Специальные кнопки:** переход к окну или подробной статье энциклопедии.

### Исполнители, Альбомы, Жанры и Композиторы
В данных разделах фонотека группируется по соответствующим метаданным аудиофайлов.
- **Исполнители:** Группировка производится на основе тега исполнителя альбома (`album artist`) или исполнителя (`artist`). Выбор группировки можно изменить в [окне настроек](#настройки-программы) на вкладке предпочтений. От выбора типа группировки будет зависеть способ навигации по исполнителям.
- **Альбомы:** Вкладка агрегирует все альбомы фонотеки.
- **Жанры:** Разделение музыки по музыкальным стилям.
- **Композиторы:** Если в метаданных композиций указан композитор — он автоматически отобразится на данной вкладке (полезно для классической музыки или саундтреков, а также для поиска авторов популярных композиций).

### Все композиции
Специальный раздел для просмотра всей музыкальной базы в виде списков, разделенных по группам (по алфавиту или периодам в зависимости от сортировки).
- В верхней части страницы располагается **панель быстрой навигации** для переключения между группами.
- Композиции сгруппированы в виде карточек, соответствующих их альбомам.

### Списки воспроизведения
Если в системной папке музыки пользователя (например, `C:\Users\user_name\Music\Playlists`) присутствуют файлы плейлистов (например, форматов `.m3u` или `.m3u8`), Виниллер автоматически их проиндексирует и отобразит в этом разделе.
- **Генерация обложки:** Если музыка из списка воспроизведения находится в фонотеке, то карточки списков воспроизведения сгенерируют мини-обложку из первой композиции с обложкой, входящей в список.
- **Краткая информация:** Если музыка из списка воспроизведения находится в фонотеке, то на карточке отображается общее количество треков в плейлисте и его суммарная продолжительность.
- **Управление:** Плейлисты можно запускать целиком или открывать их для просмотра содержимого и управления порядком с помощью перетаскивания.
- **Импорт в фонотеку:** Если музыка из списка воспроизведения НЕ состоит в фонотеке, то из контекстного меню списка воспроизведения можно выбрать действие импорта папок с музыкой, содержащихся в плейлисте.

#### Создание списков воспроизведения и добавление в список музыки
Для создания списка или добавления композиций, доступны следующие действия:
- **Из контекстного меню композиции:** можно добавить в существующий список или создать новый (в очереди воспроизведения или в основном интерфейсе проигрывателя);
- **Из меню дополнительных действий [очереди воспроизведения](#очередь-воспроизведения):** можно сохранить новый список воспроизведения, который будет состоять из композиций находящихся в очереди на момент создания.
- **Путем перетаскивания:** композицию из очереди воспроизведения можно перетянуть на карточку списка воспроизведения или, если список воспроизведения открыт, перетянуть на желаемую позицию в списке.

### Папки
Раздел предназначен для пользователей, предпочитающих классическую навигацию по файловой системе вместо сортировки по метаданным.
- На главной странице вкладки отображаются корневые директории с музыкой, которые вы добавили в фонотеку.
- Поддерживается глубокая навигация: вы можете заходить внутрь вложенных папок точно так же, как в системном проводнике.
- Папки, содержащие аудиофайлы, будут выглядеть как карточки альбомов со списком композиций.
- Запуск папки через кнопку "Play" автоматически собирает все поддерживаемые аудиофайлы внутри неё (включая вложенные папки) и добавляет их в очередь воспроизведения.

## Поиск по фонотеке

[![Vinyller](assets/main_wnd/vinyller_search_lyrics.png)](assets/main_wnd/vinyller_search_lyrics.png)

Глобальная строка поиска, расположенная в верхней части окна программы, позволяет мгновенно находить музыку по всей вашей локальной базе. Поиск работает в реальном времени: результаты начинают появляться по мере ввода текста (с небольшой задержкой для предотвращения зависаний), без необходимости нажимать клавишу `Enter`. Если очистить строку поиска, проигрыватель скроет результаты поиска.

### Режимы поиска
С помощью специальной кнопки-фильтра рядом со строкой поиска можно сузить область поиска, выбрав конкретную категорию:
- **Искать везде:** Поиск по всем доступным метаданным одновременно (значение по умолчанию).
- **Исполнители, Альбомы, Жанры, Композиторы:** Целенаправленный поиск только по указанным сущностям.
- **Композиции:** Поиск с отображением результатов в виде карточек альбомов.
- **Списки воспроизведения:** Поиск по названиям сохраненных файлов плейлистов.
- **Избранное:** Поиск только среди элементов, которые находятся в списке избранного.
- **Текст песни:** Режим, который ищет совпадения непосредственно *внутри текстов песен*, сохраненных в метаданных аудиофайлов. В результатах поиска будут отображаться специальные карточки со строками из песни, где было найдено совпадение.

### Отображение результатов и навигация
Интерфейс поиска динамически подстраивается под тип искомых данных:
- **Управление видом:** В зависимости от выбранного режима поиска, результаты можно отображать в виде списка («Все композиции»), сетки, крупной или мелкой плитки. Переключение осуществляется с помощью кнопки вида в панели поиска.
- **Вложенная навигация:** Клик по найденному исполнителю, жанру или композитору в результатах поиска откроет подробную страницу с его альбомами прямо внутри вкладки поиска. Вернуться к общему списку результатов можно с помощью кнопки <kbd>←</kbd> (Назад).

### Быстрые действия
Когда поиск возвращает список композиций (или при поиске «Искать везде»), на панели управления поиском появляются дополнительные кнопки действий:
- **Воспроизвести все:** Запускает воспроизведение всех найденных результатов, заменяя текущую очередь.
- **Встряхнуть и воспроизвести:** Физически перемешивает найденные результаты в случайном порядке, добавляет их в очередь и запускает воспроизведение.

Любая карточка или трек в результатах поиска поддерживает вызов стандартного [Контекстного меню](#контекстное-меню-и-действия) по клику правой кнопкой мыши.

## Рейтинг и чарты
Данный раздел описывает внутреннюю логику обработки статистики воспроизведения, правила начисления рейтингов и механизмы 
формирования чартов. Сформированная статистика обрабатывается и хранится **только локально** в папке `.vinyller` (`VINYLLER_FOLDER`) в каталоге пользователя.
- **Включение и отключение:** В окне настроек можно включить или отключить сбор статистики. При отключении кнопка «Чарты» скрывается из главного меню.
- **Глобальный сброс:** Вызывается из настроек и позволяет обнулить всю историю. Вы можете выбрать: удалить только текущую статистику или очистить её вместе с ежемесячными архивами.
- **Точечный сброс:** Если рейтинг конкретного элемента стал неактуален, вы можете обнулить его через контекстное меню (клик правой кнопкой мыши по карточке или треку). Проигрыватель предложит выбор: сбросить рейтинг только за текущий месяц или удалить его полностью за все время.

### Вкладка чартов
Вкладка чартов агрегирует собранную статистику и выводит её через систему карточек и списков, разделенных на категории: 
Топ композиций, альбомов, исполнителей, жанров и композиторов, а также архив чартов.

[![Vinyller](assets/charts/charts.png)](assets/charts/charts.png)

- **Временные периоды:** С помощью специальной кнопки в верхней панели управления можно переключать отображение данных между срезами «Текущий месяц» и «За все время».
- **Сортировка и отображение:** При детальном просмотре категории (по кнопке «Смотреть все») элементы можно сортировать по рейтингу (возрастание/убывание), по алфавиту (А-Я, Я-А) или по году выпуска (для альбомов). Также доступно изменение вида: списком, крупной или мелкой плиткой.

### Формирование рейтинга

#### Правила учета прослушиваний
Чтобы прослушивание было засчитано и элементу были начислены рейтинговые баллы, воспроизведение должно удовлетворять строгому набору параметров. 
Алгоритм (`PlaybackStatsHandler`) использует эти правила для валидации и защиты от накруток (например, при быстром переключении треков):
- **Базовый порог длительности (`STATS_AWARD_PERCENTAGE`):** По умолчанию прослушивание засчитывается при непрерывном воспроизведении `0.5` (50%) от общей длительности композиции.
- **Абсолютный минимум (`STATS_AWARD_MIN_S`):** Независимо от процента, трек должен воспроизводиться не менее `15` секунд. Если композиция короче, чем `STATS_AWARD_MIN_S`, то рейтинг ей не будет начисляться. Это сделано для того, чтобы избежать попадания музыкальных заполнителей и прочих коротких композиций в статистику чартов.
- **Ограничение для длинных треков (`STATS_AWARD_CAP_S`):** Для продолжительных композиций (аудиокниги, подкасты и т.п.) установлен верхний порог. Прослушивание гарантированно засчитывается после `240` секунд (4 минут) воспроизведения, даже если это меньше 50% длительности.

#### Распределение рейтинга и начисление баллов
При успешном зачете прослушивания алгоритм (`PlaybackStatsHandler`) распределяет баллы между метаданными композиции. 
Для защиты от искусственной накрутки (например, при зацикливании одного трека) используется каскадная система с проверкой контекста:
- **Базовое начисление:** Воспроизводимому треку всегда безусловно начисляется `+1` балл за каждое валидное прослушивание.
- **Осознанный выбор (`conscious_choice_data`):** Если воспроизведение запущено целенаправленно со страницы конкретной сущности (альбома, артиста, жанра, композитора), алгоритм начисляет ей бонусные `+2` балла. Если точечно был запущен конкретный трек, он получает дополнительный `+1` балл.
- **Защита от накрутки альбомов:** Связанный альбом получает `+1` балл **только в том случае**, если он сменился по сравнению с предыдущим прослушанным треком (сверка с `last_played`). Правило применяется, если альбому **еще не был начислен** бонус за осознанный выбор.
- **Универсальное начисление исполнителям:** Алгоритм собирает всех уникальных участников композиции из всех доступных тегов (основной исполнитель, исполнитель альбома, приглашенные артисты из тега `artists`). Вычисляется разница между исполнителями текущего и прошлого трека, и `+1` балл начисляется только **новым** артистам в этой цепочке. Это обеспечивает честный учет всех участников-исполнителей и защищает от накрутки при повторе.
- **Динамические теги (Жанры и Композиторы):** Аналогично исполнителям, алгоритм вычисляет разницу между тегами текущего и прошлого трека. Каждому **новому** жанру или композитору начисляется `+1` балл. Это правило работает непрерывно как при осознанном запуске, так и при фоновом воспроизведении в общей очереди (при условии, что сущность ранее не получила бонус за осознанный выбор).

#### Фоновое сохранение данных статистики
Для оптимизации работы приложения и минимизации нагрузки на диск (I/O) во время прослушивания музыки, статистика сохраняется асинхронно:
- **Интервал проверки (`STATS_POLL_INTERVAL_S`):** Фоновый поллер (таймер) просыпается каждые `30` секунд и проверяет наличие новых несохраненных данных. Если проигрыватель в этот момент находится на паузе или остановлен, данные записываются на диск.
- **Интервал принудительного сохранения (`STATS_SAVE_TRIGGER_INTERVAL_M`):** Если музыка играет непрерывно, откладывать сохранение бесконечно нельзя. Поэтому накопленные данные автоматически сбрасываются на диск каждые `10` минут.
- **Сохранение при закрытии:** При штатном закрытии приложения Виниллер форсирует немедленное сохранение всей статистики, оставшейся в оперативной памяти.

### Формирование архива чартов
[![Vinyller](assets/charts/charts_archive.png)](assets/charts/charts_archive.png)
Алгоритм менеджера фонотеки `LibraryManager` автоматически определяет смену календарного месяца. В этот момент происходит подведение итогов:
* Виниллер берет **Топ-50** самых популярных элементов из каждой категории и сохраняет их в специальный неизменяемый архив.
* Записи прошлых месяцев доступны для просмотра в отдельном блоке навигации в самом низу вкладки «Чарты».

## История воспроизведения
История воспроизведения музыки находится на отдельной вкладке основной панели навигации. 
В настройках программы можно выбрать поведение истории воспроизведения:
- **Не запоминать историю воспроизведения** — очищает историю воспроизведения и отключает видимость вкладки истории;
- **Запоминать для текущего сеанса** — хранит историю воспроизведения до закрытия проигрывателя;
- **Запоминать последние `HISTORY_LIMIT` композиций** — хранит историю между сессиями для последних `HISTORY_LIMIT` (по умолчанию 1000) записей. Опция включена по умолчанию.

**Уникальные треки в истории**
В настройках доступна опция записи в историю только уникальных композиций. Если опция включена, то предыдущая композиция в истории поднимется в верх списка вместо того, чтобы записываться несколько раз.

**Как композиция попадает в историю?**
Трек отобразится во вкладке недавней истории прослушиваний через `HISTORY_MIN_S` (по умолчанию 5 секунд) после начала воспроизведения.

# Управление проигрывателем и действия

Этот раздел содержит описание основных доступных действий и методов управления воспроизведением проигрывателя Виниллер.

## Панель управления воспроизведением
Панель управления воспроизведением предоставляет доступ ко всем основным функциям контроля над музыкой и располагается 
в нижней части окна. Панель динамически адаптируется под выбранный режим интерфейса (Основной, Винил, Мини).

### Информация о композиции
- **Обложка альбома:** При наведении курсора на обложку её можно увеличить для детального просмотра (обложка откроется в основном окне проигрывателя).
- **Название трека:** Если название слишком длинное, оно аккуратно сокращается.
- **Исполнитель, Альбом, Год и Жанры:** Все эти элементы являются кликабельными. Нажатие на них осуществит быстрый переход к соответствующей странице в вашей фонотеке.
    * *Примечание:* Если у композиции указано несколько исполнителей или жанров, то при клике на выбранный элемент откроется выпадающее меню с возможностью выбрать конкретного артиста или жанр для перехода. Если в настройках включена группировка по «Исполнителю альбома» (Album Artist), переход будет осуществлен к исполнителю альбома без возможности выбора исполнителя.

### Элементы управления
- **Полоса прогресса воспроизведения:** Отображает текущее время и общую длительность трека. Позволяет быстро перематывать композицию кликом по области на полосе прогресса.
- **Кнопки навигации:** Предыдущий трек, Воспроизведение/Пауза, Следующий трек.
- **Перемешать (Shuffle):** Включает случайный порядок воспроизведения треков до момента отключения функции. В отличие от функции `Shake and Play`, не перемешивает треки физически в очереди воспроизведения.
- **Повтор (Repeat):** Имеет три состояния — повтор выключен, повтор всей очереди, повтор одного трека.

### Дополнительные инструменты
В правой части панели собраны инструменты для управления звуком и интерфейсом:
- **Текст песни <kbd>la</kbd>:** Переключает боковую панель очереди воспроизведения в режим отображения текста текущей песни (кнопка активна только если текст присутствует в метаданных файла).
- **Добавить в избранное:** Быстрое сохранение текущего трека в ваш список избранных композиций.
- **Громкость:** Включает кнопку быстрого отключения звука (Mute) и ползунок громкости. 
  * *В режиме «Винил» громкость скрыта в компактном всплывающем окне, которое появляется при нажатии на иконку динамика.*
- **Показать/Скрыть очередь:** Управляет видимостью правой панели с текущей очередью воспроизведения.
- **Меню опций («...»):** Доступно в компактных режимах. Позволяет скрыть панель управления, включить закрепление окна «Поверх всех окон» или перейти в Ультра-компактный режим. Закрепление окна поверх других окон на ОС Linux (Wayland и т.п.) недоступно, т.к. закреплением окон оперирует пользователь в ручном режиме.

## Очередь воспроизведения
Очередь воспроизведения располагается на выдвижной панели справа и показывает список композиций, которые будут 
проиграны следующими. Панель совмещает в себе функционал управления текущим списком очереди и просмотра текстов песен.

[![Vinyller](assets/main_wnd/vinyller_artist_albums.png)](assets/main_wnd/vinyller_artist_albums.png)

### Шапка очереди
- **Источник:** Иконка и текстовая подсказка показывают, откуда играет музыка (например, конкретный альбом, плейлист, папка или «Моя волна»).
- **Статистика:** Общее количество треков в очереди и их суммарная продолжительность.

### Управление списком
- **Изменение порядка:** Вы можете менять треки местами с помощью простого перетаскивания (Drag-and-Drop).
- **Быстрый запуск:** Двойной клик по любому треку немедленно начнет его воспроизведение.
- **Контекстное меню:** Клик правой кнопкой мыши по треку открывает стандартное [меню действий](#контекстное-меню-и-действия) (добавить в избранное, перейти к артисту, редактировать теги и т.д.).

### Дополнительные действия
В шапке очереди находится кнопка меню ( <kbd>...</kbd>), открывающая меню настройки внешнего вида и управления списком:
- **Компактный вид:** Уменьшает высоту строк списка треков для отображения большего количества информации на экране.
    * *Скрыть имя исполнителя:* Доступно только в компактном виде. Оставляет только названия треков.
- **Показывать обложку:** Включает или отключает миниатюры обложек рядом с каждым треком в очереди.
- **Автовоспроизведение при добавлении в очередь:** Если опция включена, треки, добавленные в очередь, начнут играть автоматически (если до этого воспроизведение было остановлено).
- **Встряхнуть очередь (Shake Queue):** Моментально физически перемешивает все треки в текущем списке в случайном порядке. [*Не является аналогом функции `Shuffle`!*](#панель-управления-воспроизведением)
- **Очистить очередь:** Полностью удаляет все треки из списка.
- **Сохранить как плейлист:** Позволяет сохранить текущий список композиций из очереди в отдельный [файл плейлиста](#списки-воспроизведения).
- **Сформировать микстейп:** Экспортирует текущую очередь в виде физической папки с файлами (подробнее см. раздел [«Формирование микстейпов»](#формирование-микстейпов)).

### Просмотр текста песни

[![Vinyller](assets/main_wnd/vinyller_artist_albums_album_lyrics.png)](assets/main_wnd/vinyller_artist_albums_album_lyrics.png)

Если нажать на кнопку текста песни <kbd>la</kbd> в панели управления или в строке трека (или выбрать соответствующий пункт в контекстном меню трека), панель очереди переключится в режим просмотра текста:
* На панели отобразится обложка, название трека и исполнитель.
* Кнопки <kbd>А-</kbd> и <kbd>А+</kbd> в шапке позволяют уменьшить или увеличить размер шрифта для комфортного чтения.
* Для возврата к списку треков необходимо нажать кнопку <kbd>×</kbd> (Закрыть) или повторно нажать на иконку текста песни в панели управления воспроизведением.

## Контекстное меню и действия
Контекстное меню предоставляет быстрый доступ к управлению воспроизведением, фонотекой и отдельными композициями. Меню вызывается нажатием правой кнопки мыши по карточкам (альбомов, исполнителей, плейлистов) или строкам треков в любом разделе программы.

[![Vinyller](assets/main_wnd/vinyller_artist_albums_context_menu.png)](assets/main_wnd/vinyller_artist_albums_context_menu.png)

### Основные действия
В зависимости от выбранного элемента (композиция, альбом, исполнитель, композитор, список воспроизведения, папка, список избранных композиций, список истории воспроизведения и т.д.) набор действий в меню может немного отличаться, но базовые функции включают:
- **Воспроизвести:** Заменяет текущую очередь выбранными треками и сразу начинает воспроизведение.
- **Перемешать и воспроизвести (Shake and Play):** Случайным образом физически перемешивает композиции выбранного элемента (например, альбома или плейлиста) при добавлении в очередь и запускает воспроизведение. [*Не является аналогом функции `Shuffle`!*](#панель-управления-воспроизведением)
- **Воспроизвести следующим:** Добавляет выбранные композиции в очередь сразу после текущего играющего трека.
- **Добавить в очередь:** Отправляет композиции в самый конец текущей [очереди воспроизведения](#очередь-воспроизведения).
- **Показать текст песни:** Вызывает отображение [текста песни](#просмотр-текста-песни) выбранной композиции (если текст присутствует в метаданных).
- **Сбросить статистику:** Вызывается только на вкладке [чартов](#рейтинг-и-чарты). Позволяет обнулить рейтинговые баллы элемента за текущий месяц или полностью за все время.
- **Добавить в избранное / Удалить из избранного:** Добавляет или убирает элемент из [списка избранного](#добавление-музыки-в-избранное).
- **Быстрое добавление в список воспроизведения:** Добавляет или убирает элемент из сохраненных файлов [списков воспроизведения](#списки-воспроизведения).
- **Быстрый переход к исполнителю, альбому, жанру, композитору:** вызывает переход на соответствующую вкладку с открытием страницы выбранной сущности.
- **Быстрый переход к альбомам `YYYY` года выпуска:** вызывает переход на вкладку альбомов с вызовом сортировки по дате выпуска и автопрокруткой к выбранному году.
- **Найти в интернете...:** Вызывает список доступных поисковых служб, выбор службы открывает соответствующий веб-ресурс с поисковым запросом, соответствующим названию выбранной сущности.
- **Редактировать метаданные:** Открывает окно [универсального редактора метаданных](#редактор-метаданных) для изменения информации о композиции или альбоме. Редактирование метаданных недоступно для монолитных FLAC+CUE альбомов.
- **Открыть энциклопедию:** Открывает окно [энциклопедии](#энциклопедия) со статьей выбранной сущности.
- **Показать в папке:** Открывает системный файловый менеджер (Проводник, Finder и т.д.) и выделяет соответствующий аудиофайл на диске.
- **Удалить из фонотеки:** Предлагает варианты удаления: добавить файлы в черный список (чтобы скрыть из интерфейса) или навсегда удалить файлы с жесткого диска (включая связанные `.cue` файлы для виртуальных треков).

Аналогичный список действий также можно вызвать из верхней панели инструментов текущей открытой страницы, нажатием на кнопку <kbd>...</kbd>.

### Изменение обложки исполнителя, композитора, жанра
[![Vinyller](assets/main_wnd/vinyller_artist_replace_cover.png)](assets/main_wnd/vinyller_artist_replace_cover.png)

Выбор обложки возможен в верхней панели управления текущей страницей. Для этого необходимо нажать на обложку рядом названием исполнителя, композитора или жанра и из выпадающего меню выбрать новую обложку:
- **Из существующих:** Если у исполнителя, композитора или жанра присутствует несколько альбомов, то обложку основной карточки можно заменить на любую из доступных.
- **Персональная обложка:** При создании статьи энциклопедии для исполнителя, композитора или жанра можно добавить персональную обложку (например, фото исполнителя).

### Формирование микстейпов
Виниллер позволяет создавать физические сборники из вашей музыки для использования вне проигрывателя. Это удобно для быстрого переноса музыки на флешку, в телефон, автомобильную магнитолу или на международную космическую станцию без необходимости собирать файлы по всей фонотеке вручную.
* В контекстном меню списков воспроизведения, избранных треков или прямо в меню очереди воспроизведения доступна опция **«Сформировать микстейп»**.
* При её выборе Виниллер предложит указать папку на компьютере, куда будут **скопированы** все аудиофайлы, входящие в выбранный элемент с нумерацией по текущему порядку в интерфейсе.
* В созданной папке также будет создан файл списка воспроизведения.

### Быстрый экспорт композиций
Если в настройках включена опция, разрешающая экспорт файлов с помощью перетаскивания, то из очереди воспроизведения 
можно перетащить одну или несколько композиций в любую папку на диске. Композиции будут **скопированы** как отдельные файлы в выбранное расположение.

Для быстрого поиска информации о музыке в интернете предусмотрена интеграция с поисковыми службами:

- **Пользовательские ссылки:** Вы можете добавлять собственные поисковые службы (например, поиск по Spotify, Apple Music, магазинам пластинок и т.д.), используя опцию добавления новой поисковой службы.

### Перетаскивание файлов в окно программы

[![Vinyller](assets/first_start/start_dragndrop.png)](assets/first_start/start_dragndrop.png)

Виниллер поддерживает добавление музыки с помощью перетаскивания. При перетягивании файла в окно проигрывателя интерфейс отобразит три области:
- **Добавление в фонотеку:** перенесенные файлы или папки будут добавлены в фонотеку проигрывателя;
- **Добавление в очередь воспроизведения:** перенесенные файлы будут добавлены в конец текущей очереди воспроизведения;
- **Замена очереди воспроизведения:** перенесенные файлы заменят текущую очередь воспроизведения.

## Режимы окна проигрывателя
Виниллер имеет 3 режима окна. Переключение между режимами осуществляется с помощью кнопок в интерфейсе 
или [горячей клавиши](#горячие-клавиши) <kbd>C</kbd>

### Основной режим

[![Vinyller](assets/wnd_modes/modes_main.png)](assets/wnd_modes/modes_main.png)

Классический режим окна подходит для полноценного управления фонотекой и воспроизведением. 

Переключение между основным и «виниловым» режимами осуществляется с помощью кнопки изменения 
режима на левой панели навигации рядом с кнопкой настроек или [горячей клавиши](#горячие-клавиши) <kbd>C</kbd>

### Режим «Винил»

[![Vinyller](assets/wnd_modes/modes_vinyl.png)](assets/wnd_modes/modes_vinyl.png)

Режим окна «Винил» содержит визуализацию распаковки и проигрывания пластинки. 

- **Распаковка пластинки:** При нажатии кнопки распаковки пластинки (в левом верхнем углу при наведении курсора на область обложки) будет проиграна анимация распаковки пластинки.
- **Закрепление окна:** Окно в данном режиме можно закрепить поверх остальных окон ОС, вызвав меню действий <kbd>...</kbd> на панели управления воспроизведением. На ОС Linux (Wayland и т.п.) недоступно, т.к. закреплением окон оперирует пользователь в ручном режиме.
- **Изменение размеров:** При изменении размеров окна визуализация обложки и пластинки, а также панель управления будут адаптироваться под новый размер окна.
- **Скрытие панели управления:** Из меню действий <kbd>...</kbd> на панели управления можно вызвать скрытие панели управления.
- **Очередь воспроизведения и тексты песен:** В данном режиме также доступно отображение текущей [очереди воспроизведения](#очередь-воспроизведения) и просмотра [текста песни](#просмотр-текста-песни).
- **Стилизация:** в настройках доступна опция визуальной стилизации обложек и пластинок для данного режима.

Возврат к Основному режиму окна осуществляется с помощью кнопки в левом верхнем углу визуализации, которая появляется 
при наведении курсора на область отображения обложки.

Переключение между режимом «Винил» и «Мини» осуществляется из меню действий в панели управления в режиме винила или с помощью [горячей клавиши](#горячие-клавиши) <kbd>C</kbd>

### Режим «Мини»

[![Vinyller](assets/wnd_modes/modes_mini_vinny.png)](assets/wnd_modes/modes_mini_vinny.png)

Режим окна «Мини» всегда отображается поверх всех окон ОС. В настройках программы можно включить или отключить прозрачность окна в неактивном режиме (т.е. когда окно не в фокусе) для ОС Windows и macOS. 

- **Очередь воспроизведения и тексты песен:** В данном режиме также доступно отображение текущей [очереди воспроизведения](#очередь-воспроизведения) и просмотра [текста песни](#просмотр-текста-песни).
- **Изменение размеров:** можно изменить высоту блока очереди воспроизведения и текста песни. Если окно закреплено в нижней части экрана, то блок очереди воспроизведения будет открываться в противоположную сторону.

Возврат к режиму «Винил» осуществляется с помощью кнопки в правой части окна.

Переключение между режимом «Мини» и основным режимом осуществляется с помощью [горячей клавиши](#горячие-клавиши) <kbd>C</kbd>

## Горячие клавиши
Виниллер поддерживает горячие кнопки управления, позволяя быстро переключать треки, настраивать громкость и управлять 
интерфейсом без помощи мыши. Ниже представлен полный список доступных сочетаний:

### Управление воспроизведением
- **Воспроизведение / Пауза:** <kbd>Space</kbd> (Пробел)
- **Следующий трек:** <kbd>F</kbd> или <kbd>Ctrl</kbd> + <kbd>→</kbd> (Стрелка вправо)
- **Предыдущий трек:** <kbd>B</kbd> или <kbd>Ctrl</kbd> + <kbd>←</kbd> (Стрелка влево)
- **Перемотка трека:** Сочетания от <kbd>Ctrl</kbd> + <kbd>0</kbd> до <kbd>Ctrl</kbd> + <kbd>9</kbd> позволяют мгновенно переместиться на определенный процент длительности трека (где 0 — это начало или 0%, 1 — 10%, а 9 — 90% трека).

### Управление звуком и режимами
- **Увеличить громкость:** <kbd>+</kbd>, <kbd>=</kbd> или <kbd>Ctrl</kbd> + <kbd>↑</kbd> (громкость увеличивается с шагом 5%).
- **Уменьшить громкость:** <kbd>-</kbd> или <kbd>Ctrl</kbd> + <kbd>↓</kbd> (громкость уменьшается с шагом 5%).
- **Включить / Отключить звук (Mute):** <kbd>M</kbd>
- **Включить / Отключить перемешивание (Shuffle):** <kbd>S</kbd>
- **Переключить режим повтора:** <kbd>R</kbd>

### Навигация и интерфейс
- **Переключение режимов окна:** <kbd>C</kbd> позволяет быстро переключаться между основным режимом окна, режимом «Винил» и режимом «Мини».
- **Добавить в избранное:** <kbd>L</kbd> добавляет или удаляет текущий трек из списка избранных композиций.
- **Фокус на строке поиска:** <kbd>Ctrl</kbd> + <kbd>F</kbd> мгновенно переводит курсор в глобальную строку поиска; если в этот момент был открыт режим «Винил», Виниллер автоматически вернется к основному интерфейсу фонотеки.
- **Открыть настройки:** <kbd>P</kbd>

# Метаданные и энциклопедия

Этот раздел содержит описание основных возможностей управления метаданными и энциклопедией проигрывателя Виниллер.

## Редактор метаданных
Виниллер оснащен встроенным универсальным редактором тегов, который позволяет изменять метаданные аудиофайлов 
(ID3, Vorbis Comments, MP4, ASF) как поодиночке, так и в пакетном режиме. Вызвать редактор можно из контекстного меню трека, 
альбома, папки или выделенной группы файлов **в списке очереди воспроизведения**. Редактирование метаданных реализовано с помощью библиотеки [Mutagen](https://github.com/quodlibet/mutagen).
- При сохранении изменений Виниллер отобразит сводную таблицу с изменениями и запросит подтверждение изменений.
- Поля Исполнитель, Композитор и Жанр поддерживают списки — для ввода нескольких значений необходимо использовать точку с запятой <kbd>;</kbd>, например `Artist A; Artist B`. Виниллер корректно распознает список и позволит совершать быстрые переходы к каждому отдельному значению из списков композиций, карточек альбомов и панели управления воспроизведением. 

### Режим редактирования одного трека

[![Vinyller](assets/meta_edit/metaedit_single_track.png)](assets/meta_edit/metaedit_single_track.png)

При редактировании одного файла отобразится форма которая состоит из:
- **Обложка:** Позволяет выбрать любое изображение (кнопка «Изменить...»). Если результат не понравился, доступны кнопки отмены и повтора действия. 
- **Базовые теги:** Основные поля, такие как Название, Исполнитель, Альбом, Жанр, Год, а также номера трека и диска. Для жанров и исполнителей работает автодополнение на основе вашей фонотеки.
- **Текст песни:** Позволяет вручную вставить слова песни (при вставке форматирование автоматически очищается от тегов). 
  - **Кнопка <kbd>Найти...</kbd>** — позволяет выбрать [поисковую службу](#поисковые-службы) и перейти в интернет-браузер с заданным поисковым запросом `Lyrics {artist_title} {track_title}`.
  - **Кнопка <kbd>Скачать с LRCLIB</kbd>** — позволяет автоматически найти и загрузить текст по текущему названию трека и исполнителю, указанных в форме, если текст доступен в базе LRCLIB.
- **Расширенные теги:** Содержит техническую информацию (формат, битрейт, частота дискретизации) и расширенные теги (BPM, ISRC, Copyright, URL и др.)
  * _Примечание: список доступных дополнительных полей динамически меняется в зависимости от формата исходного файла (MP3, FLAC, MP4). Некоторые теги остаются только в режиме чтения._

### Режим редактирования альбома

[![Vinyller](assets/meta_edit/metaedit_album.png)](assets/meta_edit/metaedit_album.png)

Если вызвать изменение метаданных для альбома, редактор откроется в пакетном режиме, который будет состоять из следующих блоков:
- **Список файлов:** Содержит все треки выбранного альбома. Иконка с предупреждением укажет, в каких файлах были произведены изменения. С помощью клика правой кнопкой мыши на файле из списка можно вызвать контекстное меню и перейти к папке, содержащей файл.
- **Блок общей информации об альбоме:**
  - **Общие теги:** Поля, которые одинаковы для всего альбома — Обложка, Альбом, Исполнитель альбома и Год. 
- **Список тегов файла:**
  - **Вкладка «Базовые теги»:** Основные поля, такие как Название, Исполнитель, Альбом, Жанр, Год, а также номера трека и диска. Для жанров и исполнителей работает автодополнение на основе данных фонотеки.
    - **Кнопка <kbd>Найти...</kbd>** — позволяет выбрать [поисковую службу](#поисковые-службы)  и перейти в интернет-браузер с заданным поисковым запросом `Lyrics {artist_title} {track_title}`.
    - **Кнопка <kbd>Скачать с LRCLIB</kbd>** — позволяет автоматически найти и загрузить текст по текущему названию трека и исполнителю, указанных в форме для редактируемого трека, если текст доступен в базе LRCLIB.
  - **Вкладка «Расширенные теги»:** Содержит техническую информацию (формат, битрейт, частота дискретизации) и расширенные теги (BPM, ISRC, Copyright, URL и др.)
    * *Примечание: список доступных дополнительных полей динамически меняется в зависимости от формата исходного файла (MP3, FLAC, MP4). Некоторые теги остаются только в режиме чтения.*
- **Режим множественного редактирования:** Если в списке файлов выделить несколько треков, включится режим множественного редактирования. Если значения тегов у выбранных треков отличаются, в поле появится заглушка `<Different Values>`. Введенные в этом режиме значения будут применены ко всем выделенным файлам.

[![Vinyller](assets/meta_edit/metaedit_multi_track.png)](assets/meta_edit/metaedit_multi_track.png)

- Из контекстного меню папки (на вкладке «[Папки](#папки)»). 
- Из контекстного меню при выборе нескольких композиций в списке [очереди воспроизведения](#очередь-воспроизведения).

- **Вкладка «Базовые теги»:** 
  - **Обложка и путь к файлу:** Позволяет выбрать любое изображение (кнопка «Изменить...»). Если результат не понравился, доступны кнопки отмены и повтора действия. 
  - **Общие теги:** Основные поля, такие как Название, Исполнитель, Альбом, Жанр, Год, а также номера трека и диска.
  - **Текст песни:**
      - **Кнопка <kbd>Найти...</kbd>** — позволяет выбрать [поисковую службу](#поисковые-службы)  и перейти в интернет-браузер с заданным поисковым запросом `Lyrics {artist_title} {track_title}`.
      - **Кнопка <kbd>Скачать с LRCLIB</kbd>** — позволяет автоматически найти и загрузить текст по текущему названию трека и исполнителю, указанных в форме для редактируемого трека, если текст доступен в базе LRCLIB.

  * *Примечание: список доступных дополнительных полей динамически меняется в зависимости от формата исходного файла (MP3, FLAC, MP4). Некоторые теги остаются только в режиме чтения.*
- **Режим множественного редактирования:** Если в списке файлов выделить несколько треков, включится режим множественного редактирования. ### Интеграция поиска метаданных в Apple Music
В верхней части редактора расположена кнопка **«Поиск в Apple Music»**, которая служит для автоматического поиска и заполнения тегов:
1. При вызове поиска автоматически заполняется поле запроса из текущих значений формы и отображается результат поиска.
2. При подтверждении выбора проигрыватель автоматически скачает обложку альбома и подгрузит список всех треков альбома.
3. **Сопоставление (Apple Music Match):** Виниллер попытается автоматически связать ваши локальные файлы с треками из Apple Music на основе схожести названий или номеров треков. Вы также можете выбрать нужный трек вручную из выпадающего списка — все поля (Название, Номер трека и т.д.) заполнятся автоматически.

### Сохранение изменений
Перед окончательной записью данных в файлы Виниллер покажет окно подтверждения:
- **Таблица изменений:** Список, где наглядно показано, какое значение было до редактирования и каким станет после сохранения изменений. 
- **Быстрое сохранение:** Опция «Быстрое сохранение» доступна при редактировании отдельных композиций. Позволяет записать изменения в файлы на жестком диске, но отложить визуальное обновление интерфейса проигрывателя и данных в кэше фонотеки.

## Энциклопедия
Встроенная музыкальная энциклопедия Виниллер позволяет создавать, хранить и организовывать информацию о 
любимых исполнителях, альбомах, жанрах и композиторах. Все данные и изображения хранятся локально, 
образуя вашу персональную базу знаний, тесно интегрированную с фонотекой проигрывателя.

### Окно менеджера энциклопедии

[![Vinyller](assets/encyclopedia/ency_main_wnd.png)](assets/encyclopedia/ency_main_wnd.png)

Доступ к управлению базой знаний осуществляется через **Менеджер энциклопедии** 
(вызывается из [контекстных меню](#контекстное-меню-и-действия) или верхней панели управления 
вкладки «[Музыкальный центр](#музыкальный-центр)»). Окно разделено на две основные панели:

* **Левая панель (Навигация и поиск):** Содержит строку поиска и список всех доступных сущностей (как добавленных в энциклопедию, так и просто присутствующих в вашей локальной фонотеке). 
  * **Фильтры:** Вы можете фильтровать список по типу (Исполнители, Альбомы, Жанры, Композиторы), по статусу (Только заполненные статьи или Только пустые) и по наличию файлов в фонотеке (В фонотеке / Отсутствует).
  * **Сортировка:** Доступна сортировка по алфавиту и дате последнего изменения.
  * Цветовая индикация иконок в списке подскажет, привязан ли данный элемент к реальным аудиофайлам на вашем диске (акцентный цвет) или существует только в виде текстовой статьи (серый цвет).
* **Правая панель (Просмотр и чтение):** При выборе элемента из списка открывается полноценная карточка статьи с обложкой, описанием, ссылками и галереей. В верхней части панели расположены кнопки навигации (Назад), изменения размера шрифта для комфортного чтения (<kbd>A-</kbd> / <kbd>A+</kbd>) и кнопка перехода в режим редактирования. 

Через кнопку меню <kbd>...</kbd> рядом со строкой поиска можно создать новую статью с нуля (через специальное окно предварительной настройки с выбором типа и базовых данных), а также вызвать функции [импорта и экспорта](#импорт-экспорт-и-резервные-копии-энциклопедии).

### Добавление и редактирование статьи

[![Vinyller](assets/encyclopedia/ency_main_edit_article.png)](assets/encyclopedia/ency_main_edit_article.png)

Вызвать редактор статьи можно из Менеджера энциклопедии, либо из [контекстного меню](#контекстное-меню-и-действия) 
исполнителя, композитора, альбома или жанра в основном окне плеера. Редактор состоит из нескольких вкладок:

1. **Общие:** 
   * **Привязка к фонотеке (Library Binding):** Критически важная функция, позволяющая связать текстовую статью с реальным объектом из вашей библиотеки (например, конкретным альбомом). Если привязка активна, метаданные статьи (год, жанр, участники) будут заполнены автоматически. В этом случае у привязанного объекта в основном интерфейсе проигрывателя будет отображаться блок с информацией из энциклопедии.
   * **Название и Обложка:** Установка основного изображения статьи. Изображение может быть использовано [для установки в качестве основного](#изменение-обложки-исполнителя-композитора-жанра) для карточек фонотеки в основном окне проигрывателя.
   * **Быстрый поиск в Wikipedia:** Кнопка с иконкой лупы позволяет автоматически найти статью в Википедии (с выбором языка) и мгновенно загрузить краткое описание и заглавную фотографию.
   * **Краткое описание:** Текстовое поле для ввода основной информации (рекомендуется до 800 символов, так как этот текст будет отображаться в компактных блоках-вставках основного окна).
2. **Содержание:** Позволяет добавлять дополнительные смысловые блоки (например, «Ранние годы», «Критика», «Награды»). Каждый блок имеет свой заголовок, источник и поддерживает форматирование текста (жирный, курсив, списки). Блоки можно менять местами с помощью кнопок стрелок на элементах списка блоков.
3. **Галерея:** Сюда можно загрузить дополнительные фотографии, сканы буклетов или промо-материалы. К каждому изображению можно добавить текстовую подпись.
   * Примечание: Все добавленные изображения автоматически оптимизируются и кэшируются в системную папку проигрывателя. Если оригинальные файлы были удалены или перемещены, вы можете в один клик убрать нерабочие пути с помощью кнопки **«Очистить недоступные файлы»**.
4. **Дискография:** Доступна для Исполнителей и Композиторов. Позволяет сформировать список релизов. Вы можете воспользоваться кнопкой **«Автопоиск»** для автоматического добавления всех связанных альбомов из вашей локальной фонотеки, либо добавить отсутствующие релизы вручную. *Если вы изменили теги у файлов (переименовали альбом), Виниллер автоматически обнаружит это и предложит мигрировать энциклопедическую статью на новое имя.*
5. **Связи:** Позволяет связывать статьи между собой для быстрой перекрестной навигации. 
   * Кнопка **«Автопоиск»** проанализирует метаданные вашей музыки и автоматически предложит добавить в связи жанры, композиторов и другие логически подходящие элементы.
   * Функция **«Связать друг с другом»** устанавливает двусторонние связи между всеми добавленными в список элементами, а **«Разорвать связи»** — массово их удаляет.
6. **Ссылки:** Добавление внешних ссылок (на официальные сайты, стриминговые платформы, интернет-магазины и т.д.). Если название ссылки не задано, то при добавлении URL автоматически будет сгенерировано название ссылки на основе домена.

### Просмотр статьи
[![Vinyller](assets/encyclopedia/ency_artist_full.png)](assets/encyclopedia/ency_artist_full.png)

Помимо просмотра статей в менеджере энциклопедии, записи так же будут отображаться непосредственно в интерфейс просмотра фонотеки:
* **Компактный блок:** На страницах артистов, альбомов или жанров в самом верху отображается компактная карточка с кратким описанием. Блок можно свернуть или развернуть по клику мыши.
* **Полноэкранный просмотр:** При нажатии кнопки «Читать далее» открывается подробная страница статьи.
* **Интерактивные элементы:** 
  * Нажатие на элементы в блоке «Связанные статьи» осуществляет быстрый переход к ним. Если статья еще не создана, Виниллер предложит сделать это, автоматически заполнив известные метаданные.
  * В блоке «Дискография» отображается **статистика собранных релизов** (например, «Собрано 5 из 10»). Карточки альбомов интерактивны: если альбом есть в фонотеке, из его [контекстного меню](#контекстное-меню-и-действия) можно сразу **запустить воспроизведение** или добавить в очередь. Если альбома нет, у карточки будет пометка «Только в энциклопедии».
  * Миниатюры в галерее можно кликнуть для увеличения и просмотра в режиме полноэкранного слайд-шоу.

### Импорт, экспорт и резервные копии энциклопедии
[![Vinyller](assets/encyclopedia/ency_main_import_restore.png)](assets/encyclopedia/ency_main_import_restore.png)

Поскольку энциклопедия наполняется вручную, Виниллер предоставляет надежные механизмы для защиты ваших данных:

* **Автоматические резервные копии:** Виниллер автоматически создает и ротирует до 5 последних резервных копий текстовой базы данных (`encyclopedia.json.bak`). Если вы случайно испортили или удалили важную статью, вы всегда можете восстановить состояние базы через Менеджер энциклопедии (кнопка <kbd>...</kbd> -> Импорт -> выбор из автоматических бэкапов) или в окне [настроек программы](#настройки-программы).
* **Экспорт:** Позволяет упаковать всю вашу энциклопедию (включая все загруженные картинки, обложки галерей и тексты) в единый ZIP-архив. Отличный способ перенести базу на другой компьютер или сделать надежную ручную копию.
* **Импорт из архива:** Позволяет восстановить одну из автоматических резервных копий или импортировать сохраненную ранее энциклопедию из архива.
* **Глобальная очистка:** В [настройках программы](#настройки-программы) (вкладка «Энциклопедия») доступна функция очистки базы изображений. Программа просканирует папку с картинками и удалит те файлы, которые больше не используются ни в одной статье.

#### Разрешение конфликтов при импорте энциклопедии

[![Vinyller](assets/encyclopedia/ency_main_import_compare.png)](assets/encyclopedia/ency_main_import_compare.png)

При импорте ранее экспортированного архива Виниллер проанализирует его содержимое. Если в вашей текущей базе уже есть такие же статьи, Виниллер предложит варианты решения конфликтов: **Перезаписать все**, **Пропустить дубликаты**, **Объединить данные** или запустить **Интерактивный режим**, где вы сможете вручную подтвердить каждое изменение, сравнивая старую и новую версию статьи.

# Настройки и процессы

Этот раздел содержит описание основных настроек и процессов работы проигрывателя Виниллер.

## Настройки программы

Окно настроек разделено на вкладки, предназначенные для изменения параметров интерфейса, логики работы и управления 
локальной фонотекой. Вызов настроек осуществляется через боковую панель навигации или с помощью горячей клавиши <kbd>P</kbd>.

### Основные настройки
[![Виниллер](assets/settings/settings_general.png)](assets/settings/settings_general.png)

Раздел содержит параметры интерфейса, выбора языка и сохранения состояний между сессиями:

- **Язык:** Выбор языка интерфейса. Требует перезапуск.
- **Цветовая схема и цвет акцента:**
  * Выбор темы оформления интерфейса. Требует перезапуск.
  * Выбор акцентного цвета интерфейса. Требует перезапуск.
- **Состояние воспроизведения при закрытии программы:** 
  * Не запоминать очередь;
  * Запомнить очередь;
  * Запомнить последний трек;
  * Запомнить позицию воспроизведения последнего трека.
- **Состояние истории воспроизведения между сессиями:**
  * Не запоминать историю;
  * Запоминать для текущего сеанса;
  * Запоминать последние 1000 треков;
  * Опция сохранения в истории только уникальных треков.
- **Интерфейс и окна:** Включение запоминания последней открытой страницы при закрытии программы и сохранения размеров окна для основного и «винилового» режимов.
- **Навигационные разделители:** Включение или отключение интерактивных разделителей на корневых страницах вкладок и в разделах избранного.
- **Экспорт файлов:** Разрешение экспорта файлов перетаскиванием (drag-and-drop) из очереди воспроизведения в папки файловой системы.

Доступен выбор различных тем оформления, а также предустановленных акцентных цветов интерфейса.

#### Color Picker (Пользовательский цвет)
[![Vinyller](assets/settings/settings_general.png)](assets/settings/settings_general.png)

При выборе пункта «Пользовательский цвет...» из меню акцентного цвета отображается кнопка с HEX-кодом. 
Нажатие открывает палитру для установки произвольного цвета элементов управления с учетом контрастности фона текущей выбранной темы оформления. 

### Предпочтения
[![Vinyller](assets/settings/settings_preferences.png)](assets/settings/settings_preferences.png)

Настройки логики сортировки, воспроизведения и пользовательской навигации:

- **Иконка избранного:** Выбор графического символа для кнопки добавления в избранное.
- **Группировка исполнителей:** Выбор тега для формирования списка исполнителей (`Artist` или `Album Artist`). От выбора будет зависеть вариант навигации по исполнителям. После внесения изменений требуется запуск сканирования фонотеки.
- **Игнорировать артикли:** Пропуск артиклей при сортировке по алфавиту. Список артиклей определен как `ARTICLES_TO_IGNORE` и содержит следующий список `the, a, an, der, die, das, ein, eine, el, la, los, las, un, una, unos, unas, o, os, as, um, uma, uns, umas, le, les, une, des, il, lo, gli, uno, de, het, een, en, ett, den, det` 
- **Игнорировать регистр тега «Жанр»:** Объединение жанров с различным написанием регистра в одну группу (например, rock, ROCK и rOcK будет отображаться как Rock).
- **Считать папки с одинаковыми альбомами как разные альбомы:** При включении опции альбомы с идентичными названиями и исполнителями, расположенные в разных директориях, отображаются как отдельные релизы (на карточках появятся иконки в виде диска и номера по порядку).
- **Случайные предложения:** Отображение блока рекомендаций на страницах фонотеки. Рекомендации появляются, если в выбранной вкладке содержится более 20 элементов.
- **Автовоспроизведение:** Запуск воспроизведения при добавлении композиции в пустую очередь.
- **Стилизация обложек:** Наложение текстуры на обложки в [режиме «Винил»](#режим-винил).
- **Звуковые эффекты:** Включение фонового звука винила (для всех режимов) и звука перемотки (скретчинга пластинки в режиме «Винил»).
- **Прозрачность окна «Мини»:** Активация полупрозрачности для неактивного окна в режиме [«Мини»](#режим-мини). На ОС Linux (Wayland и т.п.) недоступно, т.к. закреплением окон оперирует пользователь в ручном режиме.

#### Изменение порядка вкладок навигации
Кнопка «Изменить порядок» открывает окно для изменения последовательности элементов бокового меню навигации. При изменении порядка вкладок соответствующим образом изменится порядок блоков на вкладке [«Музыкальный центр»](#музыкальный-центр).

## Процессы

### Обновление фонотеки
Процесс сканирования и обновления музыкальной базы выполняется в отдельном фоновом потоке, что позволяет интерфейсу проигрывателя оставаться отзывчивым во время работы. 
- При запуске обновления программа обходит все добавленные музыкальные папки в поисках файлов поддерживаемых аудиоформатов.
- Извлечение метаданных (тегов, обложек, продолжительности) осуществляется с использованием библиотеки [Mutagen](https://github.com/quodlibet/mutagen).
- Во время обработки применяются правила черного списка: файлы, папки или сущности, добавленные в исключения, игнорируются и не попадают в базу.
- После считывания данных треки автоматически группируются по исполнителям, альбомам, жанрам и композиторам.
- Итоговые результаты сканирования сохраняются в локальный кэш-файл.

### Проверка изменений в фонотеке при запуске
Виниллер умеет отслеживать внешние изменения в музыкальных папках (например, если был добавлен новый альбом в папку, присутствующую в фонотеке, или отредактированы теги сторонней программой).
- После загрузки интерфейса автоматически запускается фоновый процесс проверки изменений.
- Этот процесс сравнивает физические параметры файлов на диске (их размер и время последнего изменения) с ранее сохраненным "слепком" структуры фонотеки.
- Если обнаруживаются новые, измененные или удаленные файлы, программа подсчитывает их количество и выводит всплывающее уведомление с предложением обновить фонотеку.
- При подтверждении обновления запускается процесс "Умного обновления" (Smart Update), который точечно обрабатывает только измененные файлы, не тратя время на полное пересканирование всей фонотеки.

### Разделение альбомов на диски
Для многодисковых изданий проигрыватель обеспечивает правильную последовательность воспроизведения.
- При чтении метаданных Виниллер извлекает информацию о номере диска из соответствующих тегов (например, `discnumber`, `TPOS`, `disk`), корректно обрабатывая форматы вида "1/2".
- При отображении списка композиций внутри альбома треки автоматически сортируются по двум критериям: сначала по номеру диска, а затем по номеру трека.

### Парсинг монолитных FLAC+CUE альбомов
Виниллер поддерживает работу с едиными аудиофайлами (образами дисков), снабженными индексной картой `.cue`.
- Во время сканирования папок программа находит файлы `.cue` и анализирует их содержимое.
- Утилита парсинга поддерживает несколько кодировок (`utf-8`, `cp1251`, `latin1`), что гарантирует корректное отображение локализованных и старых файлов разметки.
- Из файла извлекаются общие данные альбома, а также информация о каждом отдельном треке: название, исполнитель, композитор, точное время начала в миллисекундах (`start_ms`) и его продолжительность (`duration_ms`).
- Распознанные композиции добавляются в фонотеку как «виртуальные треки» (`is_virtual`).
- Виртуальные треки ссылаются на один и тот же физический аудиофайл, но в интерфейсе, результатах поиска и очереди воспроизведения они отображаются и работают как полностью самостоятельные композиции.
- Виртуальные треки можно добавлять в избранное как обычные файлы.
- Виртуальные треки так же участвуют в [формировании чартов](#рейтинг-и-чарты).
- На монолитные альбомы распространяются ограничения [редактирования метаданных](#редактор-метаданных), чтобы сохранить альбомы в том виде, в котором они были реализованы.

### Кэширование внешних треков
Виниллер формирует специальный кэш (`external_tracks_cache.json`) для аудиотреков, которые **находятся за пределами основных папок музыкальной библиотеки**, отслеживаемых приложением. Этот механизм выполняет несколько важных функций:
- **Работа с плейлистами:** При загрузке или индексации плейлистов (файлы `.m3u` / `.m3u8`), содержащих ссылки на файлы, не добавленные в фонотеку, программа извлекает их метаданные и сохраняет в кэш для быстрого доступа.
- **Восстановление очереди воспроизведения:** При перезапуске приложения Виниллер мгновенно восстанавливает последнюю очередь треков, обращаясь к внешнему кэшу, если файлы из очереди не принадлежат основной фонотеке.
- **Оптимизация производительности:** Прямое извлечение тегов (название, исполнитель, продолжительность, обложка) из аудиофайлов занимает существенное время. Кэширование позволяет мгновенно подгружать эти данные при повторных обращениях к файлам.
- **Отслеживание актуальности:** При обращении к кэшу программа сверяет время последнего изменения внешнего файла на диске с сохраненным в базе. Если файл был изменен сторонней программой, метаданные считываются заново, а кэш автоматически обновляется.

## Добавление новых локализаций
Виниллер доступен на 18 языках и поддерживает легкое добавление новых переводов. 

### Требования
* **Иконка флага:** Изображение в форматах `.png`, `.svg`, `.jpg`, размером `width: 48px; height: 36px`.
* **Файл перевода:** Перевести один из существующих файлов с переводами.

#### На заметку
В файлах локализаций в разделе переводов `# --- Other / Dynamic / Not found in source files ---` содержатся ключи и переводы, которые:
- Формируются динамически в цикле;
- Переданы в функцию `translate` в качестве переменной;
- **ИЛИ** которые **не найдены** в оригинальном коде.

### Инструменты
* **Файл управления переводами `utils_translator`:** Уже содержит базу для поддержки основных европейских и азиатских языков.
* **Файл проверки `_update_translations`:** Проверит наличие переводов для **всех** ключей **относительно всего** кода и добавит пропущенные строки в виде комментариев.
* **Файл проверки `_check_line_breaks`:** Проверит наличие разрывов строк и HTML-тегов (например, `\n`, `\\n`, `<br>`, `<br/>`, `<br />`) в переводах **относительно ключей** и добавит к строкам с пропущенными разрывами соответствующие комментарии.
* **Файл проверки `_check_line_breaks_by_ref`:** Проверит наличие разрывов строк и тегов в переводах **относительно "эталонного" файла** локализации (по умолчанию `en.py`) и добавит к строкам с пропущенными разрывами соответствующие комментарии.
* **Файл тестирования `_test_translations`:** Проверит переводы на соответствие количеству форм множественного числа, указанным в `utils_translator`. Также скрипт строго проверяет сохранение **переменных форматирования** (например, `{count}`) — если переменная из ключа потеряна в переводе, тест выдаст ошибку. Опционально можно включить проверку маркеров форматирования относительно эталонного файла (по умолчанию `en.py`).

### Порядок действий
1. **Проверка совместимости:** Проверить соответствие добавляемого языка текущим правилам плюрализации. При необходимости добавить новые условия обработки форм множественного числа в **оба** метода внутри `utils_translator`: `get_expected_plural_count` (для корректной работы тестов) и `_get_plural_form` (для правильного вывода текста в интерфейсе приложения).
2. **Файл локализации:** Добавить новый файл с переводом в каталог `translations`. Название файла должно опираться на стандарт IETF, но использовать нижнее подчеркивание в нижнем регистре (например, `pt_br.py`).
3. **Проведение предварительных проверок:** Провести проверки всеми доступными инструментами ([см. раздел выше](#инструменты)).
4. **После проверок:** Повторно использовать `_update_translations` для очистки от лишних комментариев и сброса форматирования файлов локализации.
5. **Добавление иконки флага:** Добавить изображение флага в папку `assets/flags/`.
6. **Добавление языка:** Добавить новый язык в словарь языков в файле **constants**, например `"Português do Brasil": ("pt_br", "assets/flags/br.png")`. _Для названия языка использовать **самоназвание языка**, для кода языка придерживаться **IETF/ISO 639** с нижним подчеркиванием, для названия изображения с флагом — **ISO 3166**._
7. **Тестирование:** Проверить, что перевод корректно применяется в настройках программы и без ошибок отображается в интерфейсе.
